# DevOps Delivery Notes

## 1. Flow
To maintain smooth and continuous flow, work progressed through a structured Git workflow:
- `main` for production-ready code
- `develop` for integration testing
- `feature/starter-kit-files` for isolated new work

This minimized bottlenecks and allowed parallel tasks without conflicts.

## 2. Feedback
Feedback occurred through:
- A Pull Request from `feature/starter-kit-files` to `develop`
- Inline comments and review opportunities
- Visibility into commits and changes

This ensured early discovery of issues before merging.

## 3. Learning
Throughout the process, all architectural decisions—cloud model, IAM policies, subnets, and region selection—were documented clearly.  
This allows future engineers to learn from decisions rather than guess intentions.
